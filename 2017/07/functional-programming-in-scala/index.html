Functional Programming in Scala

Thu Jul 20, 2017 by author in  scala, links

I’ve just completed the first course in the coursera scala specialization series of courses, Functional Programming Principles in Scala.

I’m not going to lie, it’s a tough course that challenged my hack-it-together python background. My python workflow goes something like this:

Using sublimetext, write a function that takes some object, usually a list or dict, some strings or ints, etc., and return a dict or list, etc. Copy that function and paste it in ipython. Create some sample data, run the function on the sample data and see if it returns what I expected. Rinse and repeat. Wrap it all up in a main function that uses all the functions I’ve written to do something useful. It works. It’s not TDD or BDD, I usually don’t write test cases even though I know I should and I want to change that. That’s why I took this course: to expose myself to ideas about better ways to program (functionally) and get some familiarity with a serious IDE and testing tools (IntelliJ).

If your goals are like mine in this regard, I recommend this course.

So, what did I learn? I like scala syntax. I’m not very good at it, but I see that the language is enormously powerful and since some of my coworkers use it and are awesome enough to help me along, I’m working on porting some of the stuff I’ve done in python over to scala, taking advantage of spark (yeah, I know python speaks spark) and some Apache Streams goodness.
